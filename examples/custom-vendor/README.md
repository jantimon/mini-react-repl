# Custom vendor example

A minimal Vite app that builds its own vendor bundle (`nanoid` on top of the
React core) instead of using `mini-react-repl/vendor-default`. The
iframe-rendered REPL shows a small list whose row ids are generated by
`nanoid()`, and the seed `App.tsx` is live-editable.

## Layout

| File                             | Role                                                                   |
| -------------------------------- | ---------------------------------------------------------------------- |
| `src/vendor/vendor.ts`           | manifest declaring what's in the bundle                                |
| `package.json` `build:vendor`    | runs `repl-vendor-build src/vendor/vendor.ts`                          |
| `public/vendor/<spec>.<hash>.js` | generated JS chunks (gitignored, regenerated on install)               |
| `public/vendor/repl.types.json`  | bundled `.d.ts` payload — fetched at runtime (gitignored, regenerated) |
| `src/vendor/repl.vendor.json`    | `{ importMap }` only — bundler-imported (gitignored, regenerated)      |
| `src/App.tsx`                    | imports the import map, fetches types in parallel, mounts `<Repl/>`    |
| `src/initial/App.tsx`            | the seed source loaded as `?raw` text into the REPL                    |

A `postinstall` script runs `build:vendor` so that fresh clones populate both
output directories without a manual step. The dev/build scripts re-run it as
well.

## Run

```sh
# from the repo root, build the workspace once so dist/ exists:
pnpm install
pnpm build

# then in this directory:
pnpm dev
```

`pnpm dev` runs `build:vendor` first (refreshes `public/vendor/*` and
`src/vendor/repl.vendor.json`) and then starts Vite at http://localhost:5174.

The `.d.ts` payload (`public/vendor/repl.types.json`) is the dominant cost
— around 1 MB for `nanoid` + the React core types. It's emitted alongside
the JS chunks and the bundle JSON embeds a `typesUrl` pointer to it, so
`<Repl/>` fetches and registers the payload itself. The bundler-imported
JSON stays tiny (~400 bytes), and Monaco's IntelliSense data loads in
parallel to the app bundle rather than inflating it.

## How `vendor.ts` works

```ts
export * from 'mini-react-repl/vendor-base'; // 6 required-core specs

import * as nanoid from 'nanoid';

export { nanoid }; // import-map key 'nanoid'
```

Each exported namespace becomes one import-map entry; the source of the
`import * as` is what gets bundled. Aliasing is just a rename
(`export { lodash as 'lodash' }` would expose lodash-es as `lodash`).

## Why an entry file instead of `--packages a,b,c`?

- knip / dep-check sees real imports, no false-positive "unused" flags
- Comments live next to each entry explaining the why
- Aliasing falls out of standard ESM
- Works in monorepos with TS-source workspace packages — esbuild resolves
  them, sidestepping Node's strip-types-in-`node_modules` restriction
